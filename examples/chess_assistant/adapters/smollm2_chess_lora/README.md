# SmolLM2 Chess Assistant LoRA

This is a compact LoRA adapter for `HuggingFaceTB/SmolLM2-135M-Instruct`, trained on the engine-grounded chess assistant data in `examples/chess_assistant/data/chess_assistant_sft.jsonl`.

The adapter is not used as a standalone chess engine. It is one language agent inside the chess assistant:

1. Image/rules agents produce or validate FEN.
2. The engine agent selects a legal move using Stockfish when available.
3. The language agent explains the engine/rules context.
4. A response gate rejects repetitive or ungrounded transformer output and falls back to deterministic engine-grounded wording.

## Training Data

- Source file: `examples/chess_assistant/data/chess_assistant_sft.jsonl`
- Rows: 40
- Teacher: Stockfish through `python-chess`
- Format: chat `messages` JSONL
- Coverage: openings, tactics, king safety, endgames, defense, and pawn-structure questions

## Training Setup

- Method: LoRA SFT
- Rank: 8
- Base model: `HuggingFaceTB/SmolLM2-135M-Instruct`
- Steps: 20
- Target modules: attention and MLP projection modules
- Hardware target: laptop CPU or Apple MPS within a low-memory workflow

## Evaluation

The committed runtime was evaluated through `dataset-forge-chess eval`:

- Legal move rate: 1.0
- Move mention rate: 1.0
- Rendered board image roundtrip rate: 1.0
- Browser QA: FEN API path, image API path, no console errors, no horizontal overflow

The raw adapter is intentionally guarded at runtime. If it does not mention the engine move or becomes repetitive, the assistant uses the deterministic engine-grounded response instead.

## Intended Use

Use this adapter to improve wording and style in the chess assistant after installing the training extra:

```bash
python -m pip install -e ".[chess-train]"
dataset-forge-chess ask \
  --fen startpos \
  --use-transformer \
  --adapter-path examples/chess_assistant/adapters/smollm2_chess_lora \
  --question "What should I play and why?"
```

## Limitations

- The adapter is tiny and trained on a compact seed dataset; chess strength comes from the engine/rules layer.
- Image recognition is built for clean top-down board images generated by or visually similar to the included renderer.
- Image-only inputs cannot infer castling rights, en passant, or move counters.
